
# Projeto AGROTIS – Teste Técnico Delphi + Firebird

Este projeto foi desenvolvido como parte do processo seletivo da empresa AGROTIS. Trata-se de uma aplicação desktop em Delphi 12 com banco de dados Firebird, utilizando a biblioteca FireDAC.

---

## Estrutura do Projeto

- codigo_fonte/ – Todos os arquivos .pas, .dfm, .dpr e .dproj
- banco_de_dados/ – Contém o banco de dados AGROTISDB.FDB
- documentos/ – Enunciado do teste técnico
- dependencias/ – DLLs necessárias (fbclient.dll)
- scripts_sql/ – Scripts SQL para criação de banco e triggers

---

## Requisitos

- Delphi 12 (ou versão compatível com FireDAC)
- Firebird 2.5 ou superior
- IBExpert, DBeaver ou FlameRobin (para administração do banco)
- Windows

---

## Como Executar

1. **Restaurar o Banco**: Utilize o arquivo script_criacao_banco_agrotis.sql para criar as tabelas e a trigger no Firebird, ou use diretamente o AGROTISDB.FDB disponível em banco_de_dados/.
2. **Abrir o Projeto**: Abra ProjectAgrotis.dpr na IDE Delphi.
3. **Verifique a Conexão**: No DataModule (UnitDM), ajuste o caminho do banco em FDConnection.Params.Database se necessário.
4. **Compile e Execute**.

---

## Funcionalidades Implementadas

- Cadastro de **Clientes** com busca por CEP via [ViaCEP](https://viacep.com.br/)
- Cadastro de **Produtos** com campo de valor unitário
- Cadastro de **Pedidos** com:
  - Vinculação de cliente
  - Itens do pedido (produto, quantidade, valor)
  - Cálculo automático do total do pedido via trigger
- Estrutura MDI com navegação intuitiva
- PageControl com abas de busca e edição
- Filtros dinâmicos nos grids por nome/descrição
- Confirmação de ações como cancelamento ou exclusão

---

## Observações Técnicas

- Todas as tabelas usam INTEGER GENERATED BY DEFAULT AS IDENTITY para autoincremento
- Uso de TFDQuery para permitir JOINs e maior controle
- Trigger TRG_ATUALIZA_TOTAL_PEDIDO mantém o TOTAL_PEDIDO sempre consistente
